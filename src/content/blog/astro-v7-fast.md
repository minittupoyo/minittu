## Astro v7、速い

Astro v7にしてから、ビルドがかなり速いです。

「速い」と言っても、巨大なサービスをビルドしているわけではありません。
このブログは個人ブログです。
記事数もそこまで多くない。

それでも、毎回記事を書いて、プレビューして、ビルドして、という流れの中で速さはかなり効きます。

待ち時間が少ない。
これだけでだいぶ気分がいいです。

## このブログのビルド時間

実際にこのブログで`bun run build`を実行してみました。

結果はこんな感じ。

```text
27 page(s) built in 2.07s
```

シェルの`time`ではこうでした。

```text
bun run build  3.051 total
```

つまり、このブログ全体の静的HTML生成が、Astroの表示上では約2秒。
コマンド全体の実時間でも約3秒です。

速い。

## 何を生成しているか

このブログは、ただMarkdownをHTMLにして終わりではありません。

ビルド時にはだいたいこんなことをしています。

- 記事ページの生成
- タグページの生成
- トップページや一覧ページの生成
- 各記事のOG画像生成(Takumi)
- Markdownの変換
- Satteri向け自作プラグインの実行
- リンクカード用のOGPキャッシュ読み込み
- 画像の最適化

今回のログでは、合計27ページが生成されていました。

記事ページだけでなく、`/posts/.../og.png`のようなOG画像も生成されています。
それでも2秒ちょっとです。

個人ブログとしては十分すぎるくらい速いです。

## OG画像はTakumiで生成している

このブログでは、各記事のOG画像もビルド時に生成しています。

よくある構成だと`satori`を使うことが多いと思います。
ただ、このブログでは`satori`ではなく`takumi-js`を使っています。

実装はこんな感じです。

```ts
import { ImageResponse } from 'takumi-js/response';
```

`src/pages/posts/[slug]/og.png.ts`で記事ごとのタイトル、説明、タグ、日付を使ってOG画像を返しています。
さらに、日本語の改行位置をいい感じにするために`budoux`も使っています。

つまり、ビルド時にはHTMLだけではなく、記事ごとのOG画像もまとめて生成しています。
それ込みで2秒台なので、かなり軽いなと感じます。

OG画像生成は重くなりがちな部分だと思っていたので、ここがサクッと終わるのはかなり嬉しいです。

## Satteriも速い

Astro v7ではMarkdownプロセッサが`Satteri`になりました。

正直、最初は少し不安でした。
今まで使っていたremark系のプラグインがそのまま使えなくなったからです。

実際、このブログでは`remark-breaks`相当のものと、`remark-link-card-plus`相当のものを自作しました。

[https://blog.minittu.net/posts/satteri-plugins/](https://blog.minittu.net/posts/satteri-plugins/)

でも、移行してしまえば体感はかなり良いです。
Markdownの変換で待たされている感じがほとんどありません。

記事を増やしても、今のところビルドが重くなった感じはないです。

## リンクカードがあっても速い

このブログでは、単独行のリンクをカード化しています。

[https://astro.build/](https://astro.build/)

リンクカードはビルド時にOGPを取得するので、普通に考えると遅くなりやすい部分です。

ただ、今はメタデータや画像を`public/satteri-link-card/`にキャッシュしています。
一度取得したリンクカードは、次回以降かなり軽くなります。

その状態で2秒台なので、普段の運用ではかなり快適です。

もちろん、キャッシュを全部消して外部サイトへ取りに行くと、その分は遅くなります。
これはAstroの問題というより、ネットワーク待ちの問題ですね。

## 待たなくていいのは正義

ビルドが速いと、作業のテンポが変わります。

少し文章を直す。
ビルドする。
すぐ終わる。

この「すぐ終わる」がかなり大事です。

ビルドに何十秒もかかると、ちょっとした修正でも面倒になります。
確認するのも億劫になります。
だんだん雑になります。

でも3秒くらいなら、気軽に回せます。

「まあ一回ビルドして確認するか」が簡単にできます。
これがかなり良い。

## Nuxtのときより軽い

以前はNuxtでブログを作っていた時期もありました。

NuxtはNuxtで便利なんですが、ブログ用途だと少し重く感じることがありました。
私の使い方だと、もっと単純でいい。
Markdownを書いて、静的HTMLが出ればそれでいい。

その点、Astroはかなり合っています。

特にこのブログみたいな、ほぼ静的な個人ブログでは強いです。
余計なJavaScriptも少なくできるし、ビルドも速い。

ブログにはこういうのでいいんだよ、という感じ。

## おわり

Astro v7、めっちゃ速いです。

このブログでは、実測でこうでした。

```text
27 page(s) built in 2.07s
コマンド全体では約3.05秒
```

もちろん環境やサイトの規模によって変わります。
でも、少なくともこのブログではかなり快適です。

記事を書いて、確認して、ビルドする。
そのサイクルが軽いのは本当に大事。

Astro v7、個人ブログにはかなり良い選択肢だと思います。
